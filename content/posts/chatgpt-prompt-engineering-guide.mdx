## プロンプトエンジニアリングとは？

プロンプトエンジニアリングとは、AIに対して最適な指示を与えるための技術・手法のことです。同じ質問でも書き方を変えるだけで、回答の精度・詳しさ・使いやすさが劇的に変化します。

プログラミングのスキルは不要で、文章の書き方を工夫するだけで誰でも実践できます。

## 基本原則：良いプロンプトの5要素

### 1. 役割（ペルソナ）を与える

AIに「あなたは〇〇の専門家です」と役割を設定すると、その分野に特化した回答が得られます。

**悪い例：**
```
マーケティングについて教えて
```

**良い例：**
```
あなたはBtoBマーケティングの専門家です。
中小企業がSNSを使って新規顧客を獲得するための
具体的な戦略を3つ提案してください。
```

### 2. 出力形式を指定する

「箇条書きで」「表形式で」「300字以内で」など、欲しい形式を明示します。

```
以下の内容を、初心者向けに分かりやすく説明してください。
・専門用語は使わない
・具体例を必ず1つ入れる
・200〜300字でまとめる
```

### 3. 背景・文脈を伝える

自分の状況や目的を伝えることで、より的確な回答が返ってきます。

```
私はWebデザインを始めたばかりの初心者です。
CSSのFlexboxを使ったレイアウト作成でつまずいています。
よくあるミスと解決策を教えてください。
```

### 4. 制約条件を設定する

NGワードや避けてほしい内容を指定することで、不要な情報が入らなくなります。

```
子ども向けの読み聞かせ絵本のストーリーを考えてください。
・対象年齢：4〜6歳
・テーマ：友情
・暴力・怖い表現は使わない
・ページ数：10ページ構成
```

### 5. 例示（Few-shot）を使う

期待するアウトプットの例を見せることで、同じスタイルの回答が得られます。

```
以下の形式で商品紹介文を書いてください。

【例】
商品名：スマートウォッチX1
紹介文：忙しいビジネスパーソンの毎日をサポート。心拍数・睡眠管理・通知確認がこれ1台で完結します。

では、「ワイヤレスイヤホンPro」の紹介文をお願いします。
```

## 実践：ビジネスで使えるプロンプト集

### メール作成

```
以下の状況でビジネスメールを書いてください。
・送信者：営業担当
・相手：初めて連絡する見込み客（50代・中小企業社長）
・目的：Web制作サービスの提案アポ取得
・トーン：丁寧だが押しつけがましくない
・文字数：300字以内
```

### 議事録・要約

```
以下の会議メモを議事録形式にまとめてください。
・決定事項・アクションアイテム・担当者を明確に
・箇条書きで読みやすく整理
・次回までの期限も記載

【会議メモ】
（ここにメモを貼り付け）
```

### アイデア出し

```
30代女性向けフィットネスアプリの差別化アイデアを10個出してください。
競合：Nike Training Club、Fitbit
避けたいコンセプト：カロリー計算・体重管理
重視したい価値観：楽しさ・継続性・コミュニティ
```

## よくある失敗パターンと改善策

| 失敗パターン | 問題点 | 改善策 |
|------------|--------|--------|
| 「〇〇を教えて」だけ | 曖昧すぎる | 目的・対象・形式を追加 |
| 一度で完璧を求める | 長すぎて混乱する | 段階的に質問を重ねる |
| 専門用語を使わない | 的外れな回答になる | 業界・分野のキーワードを入れる |
| フィードバックしない | 精度が上がらない | 「もっと具体的に」「例を追加して」と返す |

## チェーン思考（CoT）テクニック

複雑な問題には「ステップバイステップで考えてください」と付け加えるだけで精度が上がります。

```
次の数学の問題をステップバイステップで解いてください。
各ステップで何をしているか説明しながら進めてください。

問題：（ここに問題を入力）
```

## まとめ

プロンプトエンジニアリングは難しいスキルではありません。今日からすぐ使える3つのポイントを覚えてください。

1. **役割を与える**：「あなたは〇〇の専門家です」
2. **出力形式を指定する**：「箇条書きで3つ」「300字以内で」
3. **背景を伝える**：自分の状況・目的・レベルを明示する

この3つを意識するだけで、ChatGPTの回答品質は大きく変わります。ぜひ今日から試してみてください。
